## 1. Socket and pid file permissions

- [x] 1.1 Satisfy Requirement: **Socket and pid file permissions** ‚Äî `DaemonServer.swift` saves the previous umask, sets `umask(0o077)` immediately before `bind()` and restores it via `defer`. Belt-and-suspenders `chmod(socketPath, 0o600)` after bind handles macOS versions that ignore umask for AF_UNIX sockets ‚Äî the explicit chmod makes the contract verifiable via `stat(2)`.
- [x] 1.2 Reject `$TMPDIR` unset scenario ‚Äî implemented as a pure function `DaemonPaths.resolveSocketDir(socketDir:env:allowUnsafe:statF:)` returning `.rejected(.tmpdirUnset, ...)` when both env TMPDIR and `--socket-dir` are absent. CLI wiring of `--socket-dir` flag deferred to Section 1.x follow-up; the resolver itself is complete and tested.
- [x] 1.3 World-writable parent rejection ‚Äî `DaemonPaths.resolveSocketDir` checks the `S_IWOTH` bit (0o002) on the candidate dir's stat result and returns `.rejected(.parentWorldWritable, ...)` unless `allowUnsafe: true` is passed. Pure function, fully tested. CLI wiring of `--allow-unsafe-socket-dir` flag deferred to Section 1.x follow-up.
- [x] 1.4 Write pid file via `open(2)` with `O_CREAT|O_WRONLY|O_EXCL` + mode `0600` ‚Äî implemented as `DaemonPaths.writePidFile(at:pid:)`. `DaemonServeLoop.start` now `unlink`s the stale pid file (already gated by `isDaemonAlive`) and calls the new helper instead of `String.write(toFile:)`. Race-free: O_EXCL fails if another process is writing simultaneously.
- [x] 1.5 Add `DaemonSocketPermissionsTests.swift` ‚Äî 12 tests covering: (a) live pid-file mode 0600 + (b) O_EXCL behavior on real filesystem, (c) TMPDIR-unset rejection (pure resolver), (d) world-writable parent rejection (pure resolver), (e) `allowUnsafe: true` bypass (pure resolver), plus extras: explicit `--socket-dir` override wins, missing-dir rejection, trailing-slash normalization, owner-only modes (0o700/0o750/0o755) accepted. Live socket-mode test (a) for the bound socket itself deferred to a daemon-integration test pass since it requires starting the actor.

## 3. Daemon log redaction

- [x] 3.1 Satisfy Requirement: **Daemon log redaction** ‚Äî new pure module `Sources/SafariBrowser/Daemon/DaemonLog.swift` with `redactParams(method:paramsJSON:logFull:)`, `truncateResult(resultJSON:logFull:)`, `formatEntry(...)`, and `isFullLoggingEnabled(env:)`. Redaction replaces `params.source` (applescript.execute) and `params.code` (Safari.js family) with `<redacted N bytes>`; truncation walks every string in result JSON and caps anything > 256 bytes with `‚Ä¶(truncated)`, UTF-8 boundary-safe (no split sequences). Wired into `DaemonServer.Instance.dispatchLine` via injectable `setLogWriter(_:logFull:)` so unit tests inject capture closures and the production CLI hooks file emission separately.
- [x] 3.2 Honor `SAFARI_BROWSER_DAEMON_LOG_FULL=1` as opt-out ‚Äî `DaemonLog.emitFullLogWarningIfNeeded(env:writer:)` writes the spec-mandated single-line warning when the env equals literal `"1"`. `DaemonServeLoop.Server.start(...)` now accepts `env:` + `stderrWriter:` injection and calls the warning emitter once at start, then propagates `logFull` to the underlying instance so any subsequent log entries skip redaction. Other truthy strings (`"true"`, `"yes"`) do NOT enable ‚Äî exact-value match per spec wording.
- [x] 3.3 Add `Tests/SafariBrowserTests/DaemonLogRedactionTests.swift` ‚Äî 18 tests covering redaction (source / code / passthrough / logFull bypass / malformed JSON safe placeholder / byte-count fidelity), truncation (short pass-through / long-string capped + marker / logFull bypass / error metadata preserved / UTF-8 boundary safety), opt-out warning (env set / env unset / env other-value / pure decision predicate), formatEntry schema, plus an integration smoke test that wires `setLogWriter` to a real `DaemonServer.Instance`, dispatches an `applescript.execute` request through a Unix-domain socket round-trip, and asserts the captured log line contains the redaction marker and method name but NOT the raw source.
