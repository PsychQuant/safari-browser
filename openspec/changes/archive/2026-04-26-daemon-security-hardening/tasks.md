## 1. Socket and pid file permissions

- [x] 1.1 Satisfy Requirement: **Socket and pid file permissions** — `DaemonServer.swift` saves the previous umask, sets `umask(0o077)` immediately before `bind()` and restores it via `defer`. Belt-and-suspenders `chmod(socketPath, 0o600)` after bind handles macOS versions that ignore umask for AF_UNIX sockets — the explicit chmod makes the contract verifiable via `stat(2)`.
- [x] 1.2 Reject `$TMPDIR` unset scenario — implemented as a pure function `DaemonPaths.resolveSocketDir(socketDir:env:allowUnsafe:statF:)` returning `.rejected(.tmpdirUnset, ...)` when both env TMPDIR and `--socket-dir` are absent. CLI wiring of `--socket-dir` flag deferred to Section 1.x follow-up; the resolver itself is complete and tested.
- [x] 1.3 World-writable parent rejection — `DaemonPaths.resolveSocketDir` checks the `S_IWOTH` bit (0o002) on the candidate dir's stat result and returns `.rejected(.parentWorldWritable, ...)` unless `allowUnsafe: true` is passed. Pure function, fully tested. CLI wiring of `--allow-unsafe-socket-dir` flag deferred to Section 1.x follow-up.
- [x] 1.4 Write pid file via `open(2)` with `O_CREAT|O_WRONLY|O_EXCL` + mode `0600` — implemented as `DaemonPaths.writePidFile(at:pid:)`. `DaemonServeLoop.start` now `unlink`s the stale pid file (already gated by `isDaemonAlive`) and calls the new helper instead of `String.write(toFile:)`. Race-free: O_EXCL fails if another process is writing simultaneously.
- [x] 1.5 Add `DaemonSocketPermissionsTests.swift` — 12 tests covering: (a) live pid-file mode 0600 + (b) O_EXCL behavior on real filesystem, (c) TMPDIR-unset rejection (pure resolver), (d) world-writable parent rejection (pure resolver), (e) `allowUnsafe: true` bypass (pure resolver), plus extras: explicit `--socket-dir` override wins, missing-dir rejection, trailing-slash normalization, owner-only modes (0o700/0o750/0o755) accepted. ## 3. Daemon log redaction

- [x] 3.1 Satisfy Requirement: **Daemon log redaction** — new pure module `Sources/SafariBrowser/Daemon/DaemonLog.swift` with `redactParams(method:paramsJSON:logFull:)`, `truncateResult(resultJSON:logFull:)`, `formatEntry(...)`, and `isFullLoggingEnabled(env:)`. Redaction replaces `params.source` (applescript.execute) and `params.code` (Safari.js family) with `<redacted N bytes>`; truncation walks every string in result JSON and caps anything > 256 bytes with `…(truncated)`, UTF-8 boundary-safe (no split sequences). Wired into `DaemonServer.Instance.dispatchLine` via injectable `setLogWriter(_:logFull:)` so unit tests inject capture closures and the production CLI hooks file emission separately.
- [x] 3.2 Honor `SAFARI_BROWSER_DAEMON_LOG_FULL=1` as opt-out — `DaemonLog.emitFullLogWarningIfNeeded(env:writer:)` writes the spec-mandated single-line warning when the env equals literal `"1"`. `DaemonServeLoop.Server.start(...)` now accepts `env:` + `stderrWriter:` injection and calls the warning emitter once at start, then propagates `logFull` to the underlying instance so any subsequent log entries skip redaction. Other truthy strings (`"true"`, `"yes"`) do NOT enable — exact-value match per spec wording.
- [x] 3.3 Add `Tests/SafariBrowserTests/DaemonLogRedactionTests.swift` — 18 tests covering redaction (source / code / passthrough / logFull bypass / malformed JSON safe placeholder / byte-count fidelity), truncation (short pass-through / long-string capped + marker / logFull bypass / error metadata preserved / UTF-8 boundary safety), opt-out warning (env set / env unset / env other-value / pure decision predicate), formatEntry schema, plus an integration smoke test that wires `setLogWriter` to a real `DaemonServer.Instance`, dispatches an `applescript.execute` request through a Unix-domain socket round-trip, and asserts the captured log line contains the redaction marker and method name but NOT the raw source.

## 4. Stale-pid file liveness detection

- [x] 4.1 **Stale-pid file liveness detection** — `DaemonPaths` gains `PidRecord` (pid, binary path, boot) Codable struct stored as JSON on disk, plus `PidReadResult` (`.ok` / `.stale(reason)` / `.absent`) sum type. New writer `writePidFile(record:at:)` uses `open(2) + O_CREAT|O_WRONLY|O_EXCL + mode 0o600`; `readPidFile(at:)` returns `.stale` for the legacy single-integer format, malformed JSON, and JSON missing required fields. Production wire-up at `DaemonServeLoop.start` swapped to `currentPidRecord()` + new write API.
- [x] 4.2 **3-check liveness** — `DaemonPaths.isProcessAlive(record:probe:)` requires ALL three: `probe.killExists(pid)`, `probe.exec(pid) == record.exec` (string equality), `abs(probe.bootTime(pid) - record.boot) <= 2s`. `PidProbe.real` binds `kill(pid,0)` (with `EPERM` accept for cross-uid case) + `proc_pidpath` + `proc_pidinfo(PROC_PIDTBSDINFO)`. `DaemonServeLoop.isDaemonAlive` updated to consume `readPidFile` + `isProcessAlive` so a recycled pid running an unrelated binary correctly surfaces as stale.
- [x] 4.3 `Tests/SafariBrowserTests/StalePidLivenessTests.swift` — 14 tests covering: (a) recycled-pid (same pid, different binary → stale), (b) binary-path-mismatch (pid alive but `/tmp/cp` not `safari-browser` → stale), (c) happy path (matching binary path + boot within ±2s → alive). Plus extras: write/read round-trip, mode 0600 preservation, absent file → `.absent`, legacy single-integer format → `.stale`, malformed JSON → `.stale`, partial JSON record → `.stale`, kill probe failure → not alive (short-circuit), boot time outside ±2s → not alive, boot probe nil (proc_pidinfo failure) → not alive, currentPidRecord captures self pid + non-empty path + positive boot, real-self probe loop returns alive.

## 5. Version handshake edge cases

- [x] 5.1 **Version handshake treats dirty builds as always-mismatched** — `DaemonProtocol.Version` struct (Codable, Sendable) carries `(semver, commit, dirty, vendor)` where `Vendor` is an enum of `git|tarball|homebrew|source`. `encodeHandshake(version:)` writes the four fields under `protocol.version` as a JSON dict; `decodeHandshakeVersion(_:)` strict-decodes the dict and returns nil for any missing field or unknown vendor. Build-time metadata sourced via env vars `SAFARI_BROWSER_BUILD_{SEMVER,COMMIT,DIRTY,VENDOR}` so packaging scripts override per-distribution without touching the build system; pure resolver `resolveCurrentVersion(env:)` is the test seam.
- [x] 5.2 **Comparison rule** — `versionsMatch(server:client:)` returns true iff (a) neither side `dirty == true`, (b) `vendor` equal, AND (c) `semver`+`commit` equal. Either side dirty → mismatch regardless of commit equality (WIP daemon must be restarted); vendor mismatch → mismatch even when commits agree (toolchain drift across distributions). `DaemonClient.sendRequest` updated to consume `versionsMatch` instead of struct-equality so the asymmetric dirty rule lands. Both rules documented in the `DaemonProtocol` enum docstring.
- [x] 5.3 `Tests/SafariBrowserTests/DaemonHandshakeEdgeCaseTests.swift` — 14 tests: encode/decode round-trip preserves all 4 fields; old single-string format → nil (forward-compat); garbage / missing-fields → nil; `versionsMatch` permutations (clean identical → match; server.dirty / client.dirty / both.dirty → mismatch; vendor differ → mismatch; commit differ → mismatch; semver differ → mismatch); `resolveCurrentVersion` env-driven (defaults when env unset; explicit env overrides; dirty only on literal `"1"` — `"true"` / `"yes"` / `"on"` rejected; unknown vendor falls back to `.source`). DaemonProtocolTests legacy assertions migrated to consume the new struct shape.

## 6. Lifecycle commands bypass main-request actor

- [x] 6.1 **Lifecycle bypass at the accept layer** — `DaemonServer.Instance.serveConnection` peeks the method on each incoming line via `peekLifecycleMethod(line:)`. When the method is `daemon.status` or `daemon.shutdown`, dispatch routes to `dispatchLifecycle` which reads only Instance snapshot fields (`startedAt`, `requestCount`, `lastActivityEpoch`, `preCompiledCountSnapshot`) — never the cache actor. A long-running handler holding the cache cannot block lifecycle responses. `Server.start` no longer registers handlers for the two lifecycle methods; instead it installs a shutdown hook via `Instance.setShutdownHook(_:)` so the bypass can call back into the wrapper actor for pid + log file cleanup.
- [x] 6.2 **Cancellation envelope to in-flight clients** — Instance tracks `inFlightRequestIds: [Int32: Data]` (fd → JSON-encoded requestId snippet) via `markInFlight(fd:requestIdJSON:)` / `clearInFlight(fd:)` called around every handler dispatch in `dispatchLine`. On `daemon.shutdown`, the bypass snapshots in-flight slots BEFORE replying, then asynchronously writes `{requestId:..., error:{code:"cancelled", message:"cancelled by daemon shutdown"}}` to each fd via `encodeErrorRaw(requestIdJSON:code:message:)` (preserves Int / String / null requestId shape). `cancelled` added to `DaemonClient.Error.domainErrorCodes` so the client propagates rather than triggering `[daemon fallback: ...]` retry against a dying daemon.
- [x] 6.3 **5s watchdog force-exits if graceful path stalls** — `dispatchLifecycle` for `.shutdown` schedules `Task.detached(priority: .userInitiated) { try? await Task.sleep(for: .seconds(5)); Darwin._exit(0) }`. The graceful path (cancellation envelopes + Server.stop()) usually completes in milliseconds; the watchdog only fires when the in-process AppleScript runtime is mid-execution and blocking the actor system. `_exit(0)` skips at-exit handlers — appropriate for an emergency teardown that has already sent its responses.
- [x] 6.4 `Tests/SafariBrowserTests/DaemonLifecycleCancellationTests.swift` — 12 tests: (a) `daemon.status` returns within 500ms while a 800ms slow handler is in flight (proves bypass), (b) `daemon.shutdown` during a 2000ms slow handler triggers a `cancelled` envelope on the in-flight request, (c) `cancelled` is NOT fallback-worthy (`DaemonClient.Error.fallbackReason` returns nil for `remoteError(code: "cancelled", ...)`); plus pure tests for `peekLifecycleMethod` (matches both lifecycle methods, returns nil for normal requests / garbage / missing fields), `encodeRequestId` round-trip (Int / String / nil → "null"), `encodeErrorRaw` preserves requestId shape across the splice, and Instance's in-flight tracking round-trip (mark / snapshot / clear).
